# TimeWeb Cloud API Dashboard

Минималистичный веб-интерфейс для управления серверами в TimeWeb Cloud через API.

## Функционал

1. **Управление API токеном**
   - Сохранение токена в LocalStorage
   - Использование токена для всех запросов

2. **Получение информации**
   - Финансовая информация (баланс, валюта, история платежей)
   - Список доступных операционных систем
   - Список созданных серверов с деталями

3. **Управление серверами**
   - Просмотр списка серверов (CPU, RAM, IP-адреса, статус)
   - Удаление серверов
   - Создание новых серверов

## Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/timeweb-cloud-dashboard.git
cd timeweb-cloud-dashboard
```

2. Откройте файл `index.html` в браузере.

3. Получите API токен из [панели управления TimeWeb Cloud](https://timeweb.cloud/account/api)

4. Введите токен и нажмите "Сохранить токен"

## Использование API

### Требуемые эндпоинты

1. **Авторизация**
   - Токен получается в личном кабинете TimeWeb Cloud
   - Используется в заголовке `Authorization: Bearer <token>`

2. **Основные запросы:**
   - `GET /api/v1/account/finances` - финансовая информация
   - `GET /api/v1/os/servers` - доступные операционные системы
   - `GET /api/v1/servers?limit=100&offset=0` - список серверов
   - `POST /api/v1/servers` - создание сервера
   - `DELETE /api/v1/servers/{server_id}` - удаление сервера

### Параметры для создания сервера

```json
{
    "preset_id": 4797,
    "is_ddos_guard": true,
    "os_id": 99,
    "bandwidth": 1000,
    "name": "server-name",
    "comment": "optional comment"
}
```

## Безопасность

⚠️ **Важные предупреждения:**

1. API токен хранится только в LocalStorage браузера
2. Не передавайте токен третьим лицам
3. Для продакшена рекомендуется:
   - Добавить аутентификацию
   - Использовать HTTPS
   - Хранить токен на сервере, а не в клиенте
   - Добавить ограничения по IP
